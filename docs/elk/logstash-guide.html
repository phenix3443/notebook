2 Getting started with logstash

2.1 stashing your first event

创建测试网络： 

docker network create elk

启动容器： 

docker run -d \
--name elasticsearch \
--net elk \
-p 9200:9200 -p 9300:9300 \
-e "discovery.type=single-node" \
elasticsearch:7.3.2

docker run -it --rm \
--name logstash \
--net elk \
logstash:7.5.1 /bin/bash

在 logstash container 中启动 logstash， 

logstash -e 'input { stdin { } } output { stdout {} }

logstash 完成一系列启动动作，打印日志到屏幕：

The stdin plugin is now waiting for input:

2.2 Parsing Logs with Logstash

使用 filebeat 监控日志，然后讲请求转发到 logstash，处理后转发到 es，通过 kinbana 进行展示。

2.2.1 配置 filebeat 讲日志发送到 logstash

docker run -it --rm \
  --net elk \
  --name=filebeat \
  --volume="$(pwd):/data" \
  docker.elastic.co/beats/filebeat:7.5.2 /bin/bash

启动 filebeat： 

./filebeat -e -c /data/filebeat.yml -d "publish"

2.2.2 为 filebeat 配置 logstash

docker run -it --rm \
--name logstash \
--volume="$(pwd):/data" \
--net elk \
logstash:7.5.1 /bin/bash

启动 logstash： 

bin/logstash -f /data/first-pipeline.conf --config.reload.automatic
